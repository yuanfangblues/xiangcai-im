 ## 功能点
 1.单聊
 
 业务流程：用户A给用户B发送消息, B收到后给A回ok
 
 细节执行过程：
 1.A接入连接层，进行鉴权验证(考虑这个放在哪一层合适)
 2.连接层连接路由层 进行内部协议转换
 3.路由层捞B用户在线状态，如果在线，则直接推送消息连接层
 4.连接层推消息给用户B
 5.如果用户不在线，则消息落盘，等B上线了，客户端主动拉消息
 
 2.群聊
